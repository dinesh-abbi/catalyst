# Local Android Build Plan

You want to build a local Android APK using the generated `android` folder, bypassing the cloud-based Expo Application Services (EAS). Here is the plan to achieve that.

## 🚀 The Easiest Way: Standalone Test Build (Install & Use Anywhere)
If you specifically just want an `.apk` file to install and test on your own Android phone immediately—without needing to be connected to your computer's Metro Bundler—you should build a **Release** APK. 

By default, React Native will automatically sign it with a dummy debug keystore, which means you can install it on your phone without doing any complex signing setups!

Run this in your terminal:
```bash
cd android
./gradlew assembleRelease
```
**Where is the APK?**
Once finished, you can find the standalone, installable APK here:
`android/app/build/outputs/apk/release/app-release.apk`

Copy that file to your phone, install, and you're good to go!

---

## Prerequisites
To build locally, you must ensure your system has the standard Android development tools installed:
1. **Java Development Kit (JDK 17)**
2. **Android Studio & Android SDK**
3. **Android NDK** (if required by your Expo SDK version)

## Step 1: Generate the `android` Folder
If you don't already have an `android` directory in your project root, or if you need to regenerate it based on your latest `app.json` configuration, you must prebuild the app.

```bash
npx expo prebuild -p android --clean
```
*(Note: `--clean` ensures a fresh native Android project is generated, wiping out any old corrupted native files.)*

## Step 2: Navigate to the Android Directory
All local build commands for Android must be run from inside the native `android` folder.

```bash
cd android
```

## Step 3: Run the Gradle Build Command
Gradle is the build system for Android. You will use the Gradle wrapper (`gradlew`) included in the `android` folder to assemble your release APK.

To generate a standard production-ready APK (which can be side-loaded onto devices):

```bash
./gradlew assembleRelease
```

*If you specifically need an `.AAB` (Android App Bundle) to upload to the Google Play Store instead of an installable APK:*
```bash
./gradlew bundleRelease
```

## Step 4: Locate the Built APK
Once the Gradle process successfully finishes (which can take several minutes), you can find your APK file at the following path:

**For APK (`assembleRelease`):**
```
android/app/build/outputs/apk/release/app-release.apk
```

**For AAB (`bundleRelease`):**
```
android/app/build/outputs/bundle/release/app-release.aab
```

You can then grab this file, move it to your phone, and install it directly!

## Important Notes on Signing
By default, `./gradlew assembleRelease` will use a debug development keystore generated by React Native. While this APK is perfectly fine to install and test on your own device, it **cannot** be uploaded to the Play Store.

To sign it properly for the Play Store entirely locally (without EAS), you need to generate a Keystore and configure your Android project to use it:

1. **Generate the Keystore:**
   Run the following command in your terminal (you can do this in the root of your project):
   ```bash
   keytool -genkeypair -v -storetype PKCS12 -keystore my-upload-key.keystore -alias my-key-alias -keyalg RSA -keysize 2048 -validity 10000
   ```
   *Remember the password you set!* Move the `my-upload-key.keystore` file to `android/app/`.

2. **Set up Gradle Variables:**
   Edit or create `android/gradle.properties` and add:
   ```properties
   MYAPP_UPLOAD_STORE_FILE=my-upload-key.keystore
   MYAPP_UPLOAD_KEY_ALIAS=my-key-alias
   MYAPP_UPLOAD_STORE_PASSWORD=your_password
   MYAPP_UPLOAD_KEY_PASSWORD=your_password
   ```

3. **Configure the App to use the Keystore:**
   Open `android/app/build.gradle`. Locate the `signingConfigs {}` block and add the release config:
   ```gradle
   signingConfigs {
       debug {
           storeFile file('debug.keystore')
           storePassword 'android'
           keyAlias 'androiddebugkey'
           keyPassword 'android'
       }
       release {
           if (project.hasProperty('MYAPP_UPLOAD_STORE_FILE')) {
               storeFile file(MYAPP_UPLOAD_STORE_FILE)
               storePassword MYAPP_UPLOAD_STORE_PASSWORD
               keyAlias MYAPP_UPLOAD_KEY_ALIAS
               keyPassword MYAPP_UPLOAD_KEY_PASSWORD
           }
       }
   }
   ```
   Then locate the `buildTypes { release { ... } }` block and change its `signingConfig`:
   ```gradle
   buildTypes {
       release {
           signingConfig signingConfigs.release
           // ... other config ...
       }
   }
   ```

4. **Build the Signed APK:**
   Run the gradle command again:
   ```bash
   cd android
   ./gradlew assembleRelease
   ```
   Your signed, production-ready APK will be at `android/app/build/outputs/apk/release/app-release.apk`.

## EAS Build Method (Cloud Build - Recommended for Expo)

If you'd prefer to build the APK in the cloud without needing to install Android Studio or the Android SDK locally, you can use Expo Application Services (EAS).

1. Open your `eas.json` file and add a `production-apk` profile (or modify an existing preview profile) to ensure it builds an `.apk` instead of an `.aab`:
```json
{
  "build": {
    "production-apk": {
      "android": {
        "buildType": "apk"
      }
    }
  }
}
```

2. Run the EAS build command for Android using this profile:
```bash
eas build -p android --profile production-apk
```

3. Once the build is successfully completed, EAS will provide a download link or a QR code. You can scan the QR code with your phone's camera to download and install the prod-level APK directly!

## Local EAS Build (No Cloud, Runs on Your Machine)
If you have your local environment set up (Android SDK, Node.js) but want the convenience of EAS without sending your code to the cloud, you can run the build entirely on your machine.

**Run this command:**
```bash
npx eas-cli build -p android --profile production --local
```
*(Note: Because this is `production` and you have `buildType: "apk"` configured properly in your `eas.json` under `production`, this will generate a production-ready `.apk` directly on your computer.)*



# Build and Run Instructions

Here are the commands to start the application in development mode and to build a production-level APK.

## 1. Development Mode (Running on Emulator/Device with Hot Reload)

First, make sure your Android device is connected or Emulator is running.

**Start the Metro Bundler:**
```bash
npm start
```

**Run the App on Android:**
Open a new terminal tab and run:
```bash
npm run android
```

## 2. Production Build (Standalone APK)

This will create a signed (debug key) release APK that can be installed on any Android device and works offline.

**Clean and Build:**
```bash
cd android
./gradlew clean
./gradlew assembleRelease
```
